🖥️ Virtual Keyboard – Экранная клавиатура 
Интерактивная экранная клавиатура для терминала, позволяющая вводить текст без физической клавиатуры.
Идеально подходит для людей с ограниченными возможностями, для демонстраций, для использования на сенсорных экранах или в ситуациях, когда физическая клавиатура недоступна.
Реализована на 7 языках программирования с единым интерфейсом и цветным выводом.

🚀 Возможности
Отображение клавиатуры – три ряда клавиш (буквы, цифры, специальные символы) в стиле QWERTY.

Ввод текста – выбор символа с помощью ввода номера клавиши (1–…).

Управление:

0 – пробел.

- – удалить последний символ (Backspace).

. – выход из программы.

9 – переключить регистр (Shift) (опционально).

8 – переключить раскладку (RU/EN) (опционально).

Цветной вывод – клавиши и текст выделены цветами для удобства чтения.

Кроссплатформенность – работает в любом терминале, поддерживающем ANSI-цвета.

Простота использования – управление осуществляется вводом номеров, без необходимости сложных библиотек.

📖 Использование
Синтаксис (единый для всех версий):

bash
<команда> [опции]
Управление
Действие	Клавиша
Выбор буквы/символа	Введите номер клавиши (см. экран)
Пробел	0
Удалить последний символ	-
Переключить регистр (Shift)	9 (если реализовано)
Переключить раскладку (RU/EN)	8 (если реализовано)
Выход	.
Примеры
bash
# Запуск экранной клавиатуры
python virtual_keyboard.py

# Нажать клавишу 12 → буква 'W' добавится в текст
# Нажать 0 → пробел
# Нажать - → удалить последний символ
# Нажать . → выход

🧠 Логика работы
Клавиатура – три ряда символов, каждый ряд имеет свой номер.

Пользователь вводит номер – программа добавляет соответствующий символ в буфер текста.

Отображение – текущий текст и клавиатура обновляются после каждого ввода.

Специальные команды – пробел, удаление, выход.

Цвет – символы на клавиатуре выделены синим, текст – зелёным.

✨ Дополнительные фичи
Поддержка переключения раскладок (RU/EN) – в некоторых реализациях (например, Python).

Поддержка регистра – переключение между верхним и нижним регистром.

Сохранение введённого текста – можно скопировать результат в буфер обмена (опционально).
